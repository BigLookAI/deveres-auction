How we got here — the work, in order

May 2026
Bidder Evaluation platform
Six-dimension deterministic scoring (relevance, spend, win-rate, recency, engagement, reliability), artist-first per-lot aggregation, AI-generated rationales with template fallback, outreach email drafter, review dashboard with decision workflow. Plus the 1953 catalogue digitisation dataset.
1 July
Evaluation tuning & neutral branding
Operator-configurable scoring weights (the ⚖ panel), rationale grounded in each bidder's strongest matching lot, a real win-rate display bug fixed, public-safe branding.
1–2 July
Contact Reconciliation engine
Blue Cubes → system-of-record preprocessing: normalisation-aware matching, NEW/IN-SYNC/UPDATE/REVIEW classification, Excel/PDF exports. Found 2,813 duplicate groups and 1,258 misfiled Eircodes in the client master.
2 July
Review workflow, staging & the split
Validated record lifecycle (state machine), SQLite staging as the only push payload, manual edit before approval, explainable matching, review rules R1–R3, role-based access, append-only audit. Products split into two front doors. Odoo importer proven end-to-end: 70 records pushed, 0 errors, idempotent.
3 July
Odoo becomes the master — spreadsheet retired
The reconciler now fetches the contact list live from Odoo and pushes back by exact database id. Four data-quality guards added from live testing (concatenated address fields, county name forms, Excel-corrupted phones, the R4 name-conflict rule). 179 tests.
6 July
Public demo & this hub
Synthetic-data demo Odoo on the server so anyone can test the full loop safely; the platform explainer document; Bidder Evaluation re-published; this page.
